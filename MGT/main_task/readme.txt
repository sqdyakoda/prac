Как собирать и запускать программу

Для сборки программы в терминале сначала перейдите в директорию проекта main-task:
    cd путь_к_директории/main_task
Скомпилируйте исходный код с помощью g++, чтобы получить исполняемый файл:
    g++ -std=c++14 main.cpp -o program
Если входные данные вводятся с клавиатуры, то для запуска программы используйте команду:
    ./program
Если же входные данные передаются из файла, используйте команду, которая запустит программу и передаст ей содержимое файла:
    ./program < путь_к_файлу_с_входными_данными
Для удобства в директории main_task уже находится файл-пример с входными данными.
Можете использовать его для демонстрации работы программы с помощью команды:
    ./program < input.txt



Описание формата входного файла

1. Файл начинается с открывающей фигурной скобкой { и заканчивается закрывающей }.
2. Между ними находятся два обязательных ключа: "towers" и "connections".
Ключи заключаются в двойные кавычки ("), после каждого ставится двоеточие (:).
3. После "towers": идет квадратная скобка [, открывающая массив.
Каждый элемент массива — это объект, описывающий вышку, в формате:
    { "id": <имя_вышки>, "coordinates": {"x": <число>, "y": <число>} }
После каждого объекта, кроме последнего, ставится запятая (,).
Массив закрывается квадратной скобкой ].
4. После секции "towers", перед секцией "connections" ставится запятая (,).
5. После "connections": идет квадратная скобка [, открывающая массив.
Каждый элемент массива — это объект, описывающий соединение, в формате:
    { "from": <имя_вышки_1>, "to": <имя_вышки_2> }
После каждого объекта, кроме последнего, ставится запятая (,).
Массив закрывается квадратной скобкой ].


Пример входных данных:
{
"towers": [
    {"id": "tower1", "coordinates": {"x": 0, "y": 0}},
    {"id": "tower2", "coordinates": {"x": 10, "y": 0}},
    {"id": "tower3", "coordinates": {"x": 10, "y": 10}}
  ],
  "connections": [
    {"from": "tower1", "to": "tower2"},
    {"from": "tower1", "to": "tower3"},
    {"from": "tower2", "to": "tower3"}
  ]
}


Примечание: Файл с входными данными быть пустым не может, то есть должны присутствовать ключевые слова "towers" и "connections".
При этом сами массивы towers и/или connections (то, что внутри квадратных скобок) могут быть пустыми, то есть могут не содержать объекты
